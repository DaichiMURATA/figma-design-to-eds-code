# Hero Block - User Story

## 概要

ランディングページの最上部に配置される、視覚的インパクトの強いHeroセクション。
訪問者に対してメインメッセージを伝え、CTAへ誘導する役割を持つ。

---

## 機能要件

### FR-001: 基本表示
- **優先度**: P0（必須）
- **内容**: 
  - 大きな見出し（H1）を表示
  - 説明文（段落）を表示
  - CTAボタンを表示
  - 背景画像またはカラー表示

### FR-002: レイアウトバリエーション
- **優先度**: P0（必須）
- **内容**:
  - Default: テキスト左寄せ、画像右配置
  - Centered: テキスト中央寄せ、画像背景
  - FullWidth: 画面幅いっぱい、画像背景

### FR-003: レスポンシブ対応
- **優先度**: P0（必須）
- **内容**:
  - Desktop (1200px+): 2カラムレイアウト
  - Tablet (768px-1199px): 1カラムレイアウト
  - Mobile (< 768px): 1カラムレイアウト、コンパクト表示

---

## インタラクション要件

### IR-001: CTAボタンホバー
- **優先度**: P1（推奨）
- **内容**:
  - マウスホバー時にボタンの色が変化
  - トランジション: 0.3s ease

### IR-002: スクロールアニメーション
- **優先度**: P2（オプション）
- **内容**:
  - ページロード時にフェードイン
  - スクロール時にパララックス効果（画像のみ）

---

## アクセシビリティ要件

### AR-001: セマンティックHTML
- **優先度**: P0（必須）
- **内容**:
  - `<h1>` タグを使用（ページに1つのみ）
  - `<p>` タグで説明文
  - `<a>` タグにrole="button"を適切に使用

### AR-002: キーボードナビゲーション
- **優先度**: P0（必須）
- **内容**:
  - Tabキーで全ての要素にフォーカス可能
  - Enterキーでボタンを実行可能

### AR-003: 色のコントラスト
- **優先度**: P0（必須）
- **内容**:
  - WCAG AA基準: コントラスト比 4.5:1 以上
  - 背景画像使用時はオーバーレイで可読性確保

### AR-004: 代替テキスト
- **優先度**: P0（必須）
- **内容**:
  - 背景画像にaria-labelを提供
  - 装飾的な画像はaria-hidden="true"

## テストシナリオ

### TS-001: Default Layout表示
- **Given**: Hero blockをDefaultレイアウトで配置
- **When**: ページを表示
- **Then**: 
  - テキストが左寄せで表示される
  - 画像が右側に表示される
  - CTAボタンが表示される

### TS-002: Centered Layout表示
- **Given**: Hero blockをCenteredレイアウトで配置
- **When**: ページを表示
- **Then**:
  - テキストが中央寄せで表示される
  - 背景画像が全面に表示される
  - オーバーレイでテキストの可読性が確保される

### TS-003: レスポンシブ表示（Mobile）
- **Given**: Hero blockを配置
- **When**: 画面幅を375pxに設定
- **Then**:
  - 1カラムレイアウトに切り替わる
  - テキストサイズが小さくなる
  - 画像が上部に配置される

### TS-004: CTAボタンクリック
- **Given**: Hero blockにCTAボタンが表示されている
- **When**: ボタンをクリック
- **Then**:
  - 指定されたURLに遷移する
  - 新しいタブで開く場合はrel="noopener"が設定される

### TS-005: キーボードナビゲーション
- **Given**: Hero blockが表示されている
- **When**: Tabキーを押す
- **Then**:
  - CTAボタンにフォーカスが移動する
  - フォーカスリングが表示される
  - Enterキーでボタンが実行される
